## 1. 필수 설치 프로그램

### Node.js (v20 LTS)

nvm(Node Version Manager)으로 설치하는 것을 권장합니다.

**Mac:**
```bash
# nvm 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 터미널 재시작 후
nvm install 20
nvm use 20
node -v  # v20.x.x 확인
```

**Windows:**
1. [nvm-windows](https://github.com/coreybutler/nvm-windows/releases) 에서 `nvm-setup.exe` 다운로드 후 설치
2. 명령 프롬프트(관리자 권한)에서:
```bash
nvm install 20
nvm use 20
node -v  # v20.x.x 확인
```

### MySQL 8.0

**Mac:**
```bash
brew install mysql
brew services start mysql
mysql_secure_installation  # 비밀번호 설정
```

**Windows:**
1. [MySQL Installer](https://dev.mysql.com/downloads/installer/) 다운로드
2. "Developer Default" 선택 후 설치
3. root 비밀번호 설정 (기억해 두기!)

**공통 — DB 생성:**
```sql
-- MySQL에 접속 후
CREATE DATABASE campus_market;
```

### MySQL Workbench (선택, 초보자 권장)
- [다운로드](https://dev.mysql.com/downloads/workbench/)
- GUI로 테이블 확인, 쿼리 실행 가능
- 터미널 SQL이 불편하면 이걸 쓰세요

### Git
- **Mac:** `brew install git` 또는 Xcode Command Line Tools에 포함
- **Windows:** [Git for Windows](https://gitforwindows.org/) 설치

### VS Code (권장 에디터)
- [다운로드](https://code.visualstudio.com/)
- 권장 확장:
  - ESLint
  - Prettier
  - Tailwind CSS IntelliSense
  - MySQL (cweijan.vscode-mysql-client2)

### Postman
- [다운로드](https://www.postman.com/downloads/)
- API 테스트용. 백엔드 개발자 필수.

## 3. 프로젝트 초기 세팅 명령어

### 프론트엔드 (client/)
```bash
# 프로젝트 생성
npm create vite@latest client -- --template react

# 디렉토리 이동
cd client

# 핵심 패키지 설치
npm install react-router-dom axios socket.io-client zustand

# TailwindCSS 설치
npm install -D tailwindcss @tailwindcss/vite

# 개발 서버 실행
npm run dev
# → http://localhost:5173 에서 확인
```

### 백엔드 (server/)
```bash
mkdir server && cd server
npm init -y

# 핵심 패키지 설치
npm install express mysql2 cors dotenv bcryptjs jsonwebtoken multer socket.io nodemailer

# 개발용 패키지
npm install -D nodemon

# package.json에 스크립트 추가 (수동)
# "scripts": {
#   "dev": "nodemon src/index.js",
#   "start": "node src/index.js"
# }

# 개발 서버 실행
npm run dev
# → http://localhost:4000 에서 확인
```

## 6. Git 브랜치 전략

### 브랜치 구조
```
main          ← 안정 버전 (직접 push 금지!)
└── develop   ← 개발 통합 브랜치
    ├── feature/auth        ← 인증 기능
    ├── feature/products    ← 물품 CRUD
    ├── feature/chat        ← 채팅 기능
    └── feature/ui-layout   ← UI 레이아웃
```

### 작업 흐름
```
1. develop에서 feature 브랜치 생성
   git checkout develop
   git pull origin develop
   git checkout -b feature/내기능

2. 작업 후 커밋
   git add .
   git commit -m "feat: 물품 등록 API 구현"

3. GitHub에 push
   git push origin feature/내기능

4. GitHub에서 Pull Request 생성 (feature → develop)

5. 팀원 1명 이상 리뷰 후 머지
```

### 커밋 메시지 규칙
```
feat: 새 기능 추가
fix: 버그 수정
docs: 문서 수정
style: 코드 포맷팅 (기능 변경 없음)
refactor: 코드 리팩토링
```

**예시:**
```
feat: 물품 등록 API 구현
fix: 로그인 시 토큰 만료 에러 수정
docs: API 명세서 업데이트
```

## 7. Git 초보자 치트시트

### 처음 한 번만 하는 설정
```bash
# 이름, 이메일 설정
git config --global user.name "내이름"
git config --global user.email "내이메일@example.com"

# 레포 클론
git clone https://github.com/팀계정/캠퍼스마켓.git
cd 캠퍼스마켓
```

### 매일 하는 작업
```bash
# 1. 최신 코드 받기
git checkout develop
git pull origin develop

# 2. 내 브랜치로 이동 (없으면 생성)
git checkout feature/내기능
# 또는 새로 만들기: git checkout -b feature/내기능

# 3. 코딩 작업...

# 4. 변경사항 확인
git status

# 5. 커밋
git add .
git commit -m "feat: 작업 내용 설명"

# 6. 푸시
git push origin feature/내기능

# 7. GitHub에서 PR 생성
```

### 자주 쓰는 명령어
| 명령어 | 설명 |
|--------|------|
| `git status` | 변경된 파일 목록 확인 |
| `git log --oneline` | 커밋 히스토리 한줄로 보기 |
| `git branch` | 현재 브랜치 확인 |
| `git stash` | 작업 중인 변경사항 임시 저장 |
| `git stash pop` | 임시 저장한 변경사항 복원 |
| `git diff` | 변경된 내용 확인 |

### 충돌(Conflict) 났을 때
```bash
# 1. develop 최신 코드를 내 브랜치에 합치기
git checkout feature/내기능
git merge develop

# 2. 충돌 파일 열어서 수동 해결
#    <<<<<<< HEAD
#    내 코드
#    =======
#    다른 사람 코드
#    >>>>>>> develop
#    → 둘 중 맞는 걸 남기고 나머지 삭제

# 3. 해결 후 커밋
git add .
git commit -m "fix: merge conflict 해결"
```

## 9. 유용한 팁

### Vite 프록시 설정 (CORS 우회)
`client/vite.config.js`:
```js
export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:4000'
    }
  }
})
```
이렇게 하면 프론트에서 `/api/products` 호출 시 자동으로 `http://localhost:4000/api/products`로 전달됩니다.

### MySQL 연결 테스트
```js
// server/src/config/db.js
const mysql = require('mysql2/promise');

const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
});

module.exports = pool;
```

### 팀원 간 DB 동기화
- `server/sql/init.sql` 파일에 테이블 생성 SQL을 관리
- 스키마 변경 시 이 파일 업데이트 + 팀 공유
- 각자 로컬에서 `mysql -u root -p campus_market < server/sql/init.sql` 실행
