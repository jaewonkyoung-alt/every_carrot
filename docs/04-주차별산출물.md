# 주차별 PPT 발표 산출물 기준

> 매주 발표에서 "이것만큼은 보여줘야 한다"는 최소 기준입니다.

---

## 1주차 발표: 기획 완료 + 프로토타입

### 필수 산출물

| # | 산출물 | 형식 | 설명 |
|---|--------|------|------|
| 1 | 프로젝트 개요 | PPT 1~2장 | 서비스 소개, 목표, 대상 사용자 |
| 2 | 기능 명세서 | PPT 1~2장 | MVP 기능 목록 (표 형태) |
| 3 | 와이어프레임 | Figma 스크린샷 | 주요 5개 화면 (메인, 상세, 등록, 채팅, 마이페이지) |
| 4 | ERD | PPT 1장 | 테이블 관계도 (MySQL Workbench 또는 draw.io) |
| 5 | API 설계 | PPT 1장 | 주요 엔드포인트 목록 표 |
| 6 | 기술 스택 | PPT 1장 | 프론트/백/DB/실시간 기술 구성도 |
| 7 | 역할 분담 + 일정 | PPT 1장 | 간트차트 또는 주차별 일정표 |
| 8 | 동작 데모 | 라이브 시연 | 로그인/회원가입 화면 (UI만이라도 OK) |

### 팁
- 와이어프레임은 예쁠 필요 없음. 레이아웃과 흐름만 보여주면 됨
- ERD는 [dbdiagram.io](https://dbdiagram.io) 쓰면 빠르게 만들 수 있음
- 1주차는 "우리가 뭘 만들 건지 + 어떻게 만들 건지"를 명확히 전달하는 게 핵심

## 2주차 발표: 핵심 기능 동작

### 필수 산출물

| # | 산출물 | 형식 | 설명 |
|---|--------|------|------|
| 1 | 진행 상황 요약 | PPT 1장 | 기능별 완료/진행중/미착수 현황 |
| 2 | CRUD 라이브 데모 | 시연 | 물품 등록 → 목록 → 상세 → 수정 → 삭제 전체 흐름 |
| 3 | API 테스트 결과 | Postman 스크린샷 | 주요 API 3~5개 요청/응답 캡처 |
| 4 | DB 데이터 확인 | Workbench 스크린샷 | 실제 DB에 데이터가 들어간 모습 |
| 5 | Git 활동 | GitHub 스크린샷 | 커밋 히스토리, PR 목록 (팀 협업 증빙) |

### 시연 시나리오 (추천)
```
1. 회원가입 → 로그인
2. 물품 등록 (사진 포함)
3. 메인 페이지에서 등록된 물품 확인
4. 카테고리 필터로 검색
5. 물품 상세 페이지 확인
```

### 팁
- 데모가 막히면 당황하지 말고 Postman으로 API 직접 보여주기
- "안 되는 것"보다 "되는 것"을 먼저 보여주기
- 진행 상황을 솔직하게 공유 (교수님들은 과정을 봄)

## 3주차 발표: 채팅 + 전체 통합

### 필수 산출물

| # | 산출물 | 형식 | 설명 |
|---|--------|------|------|
| 1 | 전체 기능 현황 | PPT 1장 | 체크리스트 형태 (✅/🔄/❌) |
| 2 | 채팅 라이브 시연 | 시연 | 브라우저 2개로 실시간 대화 시연 |
| 3 | E2E 시나리오 시연 | 시연 | 가입→물품등록→검색→채팅→거래완료 전체 흐름 |
| 4 | 반응형 데모 | 시연 | PC + 모바일 화면 비교 (개발자 도구 활용) |
| 5 | 기술적 도전 | PPT 1장 | Socket.io 구현 과정에서 겪은 이슈와 해결 방법 |

### 시연 시나리오 (추천)
```
[사용자 A - 판매자]
1. 로그인
2. 물품 등록 (전자기기 / 노트북 / 30만원)

[사용자 B - 구매자]
3. 로그인
4. 물품 검색 → 상세 페이지
5. "채팅하기" 클릭

[양쪽 동시]
6. 실시간 채팅 주고받기
7. 판매자가 "예약중"으로 상태 변경
8. 거래 완료 후 "판매완료" 변경
```

### 팁
- 채팅 시연은 브라우저 2개(Chrome + 시크릿 모드)를 화면 분할해서 보여주기
- 반응형은 Chrome 개발자 도구 → Device Toolbar로 모바일 화면 보여주기
- "기술적 도전" 슬라이드는 교수님이 좋아하는 포인트 (성장 과정 어필)

## 4주차 (최종) 발표: 완성 + 회고

### 필수 산출물

| # | 산출물 | 형식 | 설명 |
|---|--------|------|------|
| 1 | 프로젝트 개요 리마인드 | PPT 1~2장 | 서비스 소개 (1주차 업데이트 버전) |
| 2 | 최종 시연 | 라이브 시연 | 전체 기능 시연 (가장 중요!) |
| 3 | 시스템 구조도 | PPT 1장 | 프론트↔백↔DB↔Socket 전체 아키텍처 |
| 4 | 주요 기능 설명 | PPT 2~3장 | 핵심 기능별 구현 방법 간단 설명 |
| 5 | 기능 완성도 | PPT 1장 | 전체 기능 목록 + 완료/미완료 표시 |
| 6 | 팀 기여도 | PPT 1장 | 팀원별 담당 기능 + GitHub 통계 |
| 7 | 회고 | PPT 1장 | 잘한 점 / 아쉬운 점 / 배운 점 |
| 8 | 프로젝트 보고서 | 별도 문서 | 전체 개발 과정 정리 (제출용) |

### 최종 시연 시나리오
```
1. 서비스 메인 페이지 소개 (PC 화면)
2. 회원가입 (학교 이메일 인증 과정)
3. 물품 등록 (사진 업로드 포함)
4. 물품 목록 + 카테고리 필터 + 검색
5. 물품 상세 페이지
6. 1:1 채팅 (실시간)
7. 거래 상태 변경
8. 마이페이지 (프로필, 내 물품)
9. 모바일 반응형 시연
```

### 팁
- 시연용 더미 데이터를 미리 넣어두기 (빈 화면은 임팩트가 없음)
- 시연 중 에러 대비: 시연 순서대로 한 번 리허설하기
- 보고서는 깔끔한 포맷이 중요 (목차, 스크린샷 포함)

- 한 슬라이드에 핵심 1가지만
- 텍스트 최소화, 표/다이어그램/스크린샷 활용
- 시연은 별도 시간 확보 (PPT와 분리)
- Figma로 만든 와이어프레임은 그대로 캡처해서 사용 가능
