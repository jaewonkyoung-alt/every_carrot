# 기능 명세서

> 캠퍼스 전용 중고 물품 거래 서비스

## MVP 필수 기능 (Must Have)

### F1. 학생 인증 시스템 `1주차`
| 항목 | 내용 |
|------|------|
| **설명** | 학교 이메일(`@university.ac.kr`) 도메인 검증을 통한 재학생 인증 |
| **상세** | 회원가입 시 학교 이메일 입력 → 6자리 인증코드 발송 → 코드 입력 시 가입 완료 |
| **담당** | 백엔드(인증 로직 + 메일 발송), 프론트엔드(인증 UI) |

### F2. 회원가입 / 로그인 / 로그아웃 `1주차`
| 항목 | 내용 |
|------|------|
| **설명** | JWT 기반 인증. Access Token + Refresh Token |
| **상세** | - 회원가입: 이메일, 비밀번호, 닉네임, 학과<br>- 로그인: 이메일 + 비밀번호<br>- 로그아웃: 토큰 폐기 |
| **담당** | 백엔드(API), 프론트엔드(폼 UI + 토큰 관리) |

### F3. 프로필 관리 `3주차`
| 항목 | 내용 |
|------|------|
| **설명** | 사용자 프로필 조회 및 수정 |
| **상세** | 닉네임, 학과, 프로필 사진 변경. 내가 올린 물품 목록 표시 |
| **담당** | 백엔드(프로필 API), 프론트엔드(마이페이지 UI) |

### F4. 물품 등록 (CRUD) `2주차`
| 항목 | 내용 |
|------|------|
| **설명** | 중고 물품 등록/수정/삭제 |
| **상세** | - 제목, 설명, 가격, 카테고리 선택<br>- 사진 1장 업로드 (필수)<br>- 등록 후 수정/삭제 가능 (본인만) |
| **담당** | 백엔드(CRUD API + 이미지 업로드), 프론트엔드(등록 폼) |

### F5. 물품 목록 조회 + 검색 + 필터 `2주차`
| 항목 | 내용 |
|------|------|
| **설명** | 전체 물품 목록, 카테고리별 필터, 키워드 검색 |
| **상세** | - 메인 페이지에 최신순 물품 목록<br>- 카테고리: 전자기기, 도서, 의류, 생활용품, 기타<br>- 제목/설명 키워드 검색<br>- 판매중인 물품만 기본 표시 |
| **담당** | 백엔드(목록/검색/필터 API), 프론트엔드(목록 UI + 검색바) |

### F6. 물품 상세 페이지 `2주차`
| 항목 | 내용 |
|------|------|
| **설명** | 개별 물품의 상세 정보 표시 |
| **상세** | 사진, 제목, 가격, 설명, 카테고리, 판매자 정보, 등록일, 거래 상태. "채팅하기" 버튼 |
| **담당** | 백엔드(상세 API), 프론트엔드(상세 페이지 UI) |

### F7. 거래 상태 관리 `3주차`
| 항목 | 내용 |
|------|------|
| **설명** | 물품의 거래 상태를 판매자가 변경 |
| **상세** | 판매중 → 예약중 → 판매완료 (판매자만 변경 가능) |
| **담당** | 백엔드(상태 변경 API), 프론트엔드(상태 변경 UI) |

### F8. 1:1 실시간 채팅 `3주차`
| 항목 | 내용 |
|------|------|
| **설명** | 판매자-구매자 간 Socket.io 기반 실시간 채팅 |
| **상세** | - 물품 상세에서 "채팅하기" 클릭 시 채팅방 생성/입장<br>- 실시간 메시지 송수신<br>- 채팅 내역 DB 저장 (새로고침해도 유지) |
| **담당** | 백엔드(Socket.io 서버 + 메시지 저장), 프론트엔드(채팅 UI) |

### F9. 채팅방 목록 `3주차`
| 항목 | 내용 |
|------|------|
| **설명** | 내가 참여 중인 채팅방 목록 표시 |
| **상세** | 채팅방별 상대방 닉네임, 물품명, 마지막 메시지 미리보기, 시간 |
| **담당** | 백엔드(채팅방 목록 API), 프론트엔드(채팅 목록 UI) |

### F10. 반응형 웹 `전 주차 (최종 점검 4주차)`
| 항목 | 내용 |
|------|------|
| **설명** | PC/모바일 모두 사용 가능한 반응형 레이아웃 |
| **상세** | TailwindCSS 반응형 유틸리티 활용. 모바일 우선(Mobile First) 설계. 주요 브레이크포인트: 640px(sm), 768px(md), 1024px(lg) |
| **담당** | 프론트엔드 전체 |

## 있으면 좋은 기능 (Nice to Have)

> 시간 여유가 있을 때만 구현. MVP 완성 후 우선순위 순으로 도전.

| 우선순위 | 기능 | 설명 | 난이도 |
|---------|------|------|--------|
| 1 | 찜하기 (관심 목록) | 물품에 하트 표시, 마이페이지에서 관심 목록 조회 | 낮음 |
| 2 | 이미지 다중 업로드 | 물품당 최대 5장 사진 업로드 + 슬라이더 | 중간 |

| 4 | 거래 후기/평점 | 거래 완료 후 상대방 평가 (별점 + 한줄평) | 중간 |
| 5 | 최근 본 상품 | 로컬스토리지 기반 최근 조회 물품 목록 | 낮음 |

## 페이지 구성

| 페이지 | 경로 | 접근 권한 | 주요 기능 |
|--------|------|----------|-----------|
| 메인 (물품 목록) | `/` | 모두 | 물품 목록, 검색, 카테고리 필터 |
| 로그인 | `/login` | 비로그인 | 이메일/비밀번호 로그인 |
| 회원가입 | `/register` | 비로그인 | 학교 이메일 인증 + 가입 |
| 물품 등록 | `/products/new` | 로그인 | 물품 정보 입력 + 사진 업로드 |
| 물품 상세 | `/products/:id` | 모두 | 물품 정보 + 채팅하기 버튼 |
| 물품 수정 | `/products/:id/edit` | 작성자만 | 물품 정보 수정 |
| 채팅 목록 | `/chats` | 로그인 | 내 채팅방 목록 |
| 채팅방 | `/chats/:id` | 참여자만 | 실시간 1:1 채팅 |
| 마이페이지 | `/profile` | 로그인 | 프로필 수정, 내 물품 목록 |

## API 엔드포인트 목록 (초안)

### 인증
| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | `/api/auth/register` | 회원가입 |
| POST | `/api/auth/verify-email` | 이메일 인증코드 확인 |
| POST | `/api/auth/login` | 로그인 |
| POST | `/api/auth/logout` | 로그아웃 |
| POST | `/api/auth/refresh` | 토큰 갱신 |

### 사용자
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/users/me` | 내 프로필 조회 |
| PUT | `/api/users/me` | 프로필 수정 |

### 물품
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/products` | 물품 목록 (검색, 필터, 페이지네이션) |
| GET | `/api/products/:id` | 물품 상세 |
| POST | `/api/products` | 물품 등록 |
| PUT | `/api/products/:id` | 물품 수정 |
| DELETE | `/api/products/:id` | 물품 삭제 |
| PATCH | `/api/products/:id/status` | 거래 상태 변경 |

### 채팅
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/chats` | 내 채팅방 목록 |
| POST | `/api/chats` | 채팅방 생성 (물품ID + 상대방ID) |
| GET | `/api/chats/:id/messages` | 채팅 메시지 내역 조회 |

### Socket.io 이벤트
| 이벤트 | 방향 | 설명 |
|--------|------|------|
| `join_room` | Client → Server | 채팅방 입장 |
| `send_message` | Client → Server | 메시지 전송 |
| `receive_message` | Server → Client | 메시지 수신 |
| `leave_room` | Client → Server | 채팅방 퇴장 |

## DB 스키마 (ERD)

```
users
├── id            INT (PK, AUTO_INCREMENT)
├── email         VARCHAR(100) UNIQUE
├── password      VARCHAR(255) (bcrypt 해시)
├── nickname      VARCHAR(30) UNIQUE
├── department    VARCHAR(50)
├── profile_image VARCHAR(255) NULL
├── is_verified   BOOLEAN DEFAULT FALSE
└── created_at    DATETIME DEFAULT NOW()

products
├── id            INT (PK, AUTO_INCREMENT)
├── seller_id     INT (FK → users.id)
├── title         VARCHAR(100)
├── description   TEXT
├── price         INT
├── category_id   INT (FK → categories.id)
├── status        ENUM('판매중','예약중','판매완료') DEFAULT '판매중'
├── image_url     VARCHAR(255)
├── created_at    DATETIME DEFAULT NOW()
└── updated_at    DATETIME DEFAULT NOW() ON UPDATE

categories
├── id            INT (PK, AUTO_INCREMENT)
└── name          VARCHAR(20)

chat_rooms
├── id            INT (PK, AUTO_INCREMENT)
├── product_id    INT (FK → products.id)
├── buyer_id      INT (FK → users.id)
├── seller_id     INT (FK → users.id)
└── created_at    DATETIME DEFAULT NOW()

messages
├── id            INT (PK, AUTO_INCREMENT)
├── chat_room_id  INT (FK → chat_rooms.id)
├── sender_id     INT (FK → users.id)
├── content       TEXT
├── is_read       BOOLEAN DEFAULT FALSE
└── created_at    DATETIME DEFAULT NOW()

email_verifications
├── id            INT (PK, AUTO_INCREMENT)
├── email         VARCHAR(100)
├── code          VARCHAR(6)
├── expires_at    DATETIME
└── created_at    DATETIME DEFAULT NOW()
```
